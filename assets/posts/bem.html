BEM for CSS Harmony

  • by: Ali R. Amirdehi
  • date: January 11, 2024

One common issue in large-scale CSS projects is unintentional style interference. As a web application grows, different developers may be working on various components, and conflicts can arise when styles unintentionally affect each other.

What can we do to avoid this problem?
Implementing the BEM (Block Element Modifier) naming convention can significantly reduce the risk of style interference. BEM is a methodology that encourages a strict and modular approach to CSS naming.

It divides CSS classes into three categories:
1- Block: Represents a standalone entity that is meaningful on its own (e.g., header, button).
2- Element: A part of a block that has no standalone meaning and is semantically tied to its block (e.g., header_title, button_icon).
3- Modifier: A flag on a block or element, used to change appearance or behavior (e.g., button_large, header_dark).

By following the BEM convention, you create a clear and predictable structure for your styles. This reduces the likelihood of unintentional style clashes, as each component's styles are encapsulated within its block. It also enhances code readability and maintainability.

By adopting BEM, you create a modular and organized structure for your styles, making it easier to collaborate with other developers and maintain a consistent design across your web application.
